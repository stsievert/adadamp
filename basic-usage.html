Basic Usage¶

Let’s walk through PyTorch’s MNIST example. This example usage will be trimmed for brevity. The complete script can be found on GitHub at test_mnist_dask.py.

First, let’s start by creating the model.

import torch
import torch.nn as nn
from torchvision import datasets

class Net(nn.Module):
    def __init__(self):
        super(Net, self).__init__()
        self.conv1 = nn.Conv2d(1, 32, 3, 1)
        # etc; unchanged from example

     def forward(self, x):
        # etc; unchanged from example

model = Net()

This is a standard PyTorch model definition. Now, let’s create the dataset:

train_set = datasets.MNIST(...)  # internals untouched
test_set = datasets.MNIST(...)  # internals untouched

Notice a DataLoader object is not created (as with PyTorch), only the dataset that’s passed to DataLoader.

Now, let’s create our Scikit-learn wrapper with our model, loss and optimizer:

from adadamp import DaskClassifier
est = DaskClassifier(
    module=Net,
    loss=nn.NLLLoss,
    optimizer=optim.SGD,
    optimizer__lr=1e-3,
    optimizer__momentum=0.9,
    optimizer__nesterov=True,
    batch_size=256,
    max_epochs=10,
)

Again, the optimizer is a pretty standard definition. Use of SGD is recommended and backed by mathematics. Use of momentum and Nesterov acceleration is likely beneficial even though there’s no backing mathematics.

Now, we can feed our data to the fit method:

est.fit(train_set)
acc = est.score(test_set)

If desired, we can also feed our function NumPy arrays or Torch Tensors:

from sklearn.datasets import make_classification
X, y = make_classification()
est.fit(X, y)
X2, y2 = torch.from_numpy(X), torch.from_numpy(y)
est.fit(X2, y2)
